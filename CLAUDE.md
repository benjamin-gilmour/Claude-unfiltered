## Quality standards
- Every skill must have a clear trigger description
- Every skill must be tested in at least one real workflow before publishing
- README must explain what each skill does, when to use it, and how to install it
- Keep the repo clean — no experimental or broken skills in main branch

## Contribution workflow
1. New skills go in a feature branch
2. Test locally with Claude Code
3. Document in the skill's own README
4. PR to main, squash merge
